STEP 11 — RUNTIME ARCHITECTURE RULES
===============================================================================

RULE output_mode
Next.js static export (output:"export" in next.config.ts).
No server-side rendering at runtime. All pages are pre-rendered to static HTML.
Dynamic routes use generateStaticParams() seeded from SEED_* data.

RULE no_api_routes
No API routes exist. All backend access goes directly through the Supabase client.
Supabase REST + Realtime is the only backend.

RULE supabase_nullable
Supabase client returns null when NEXT_PUBLIC_SUPABASE_URL or NEXT_PUBLIC_SUPABASE_ANON_KEY are absent.
This allows static export / CI builds without secrets.
Every consumer checks for null before using the client.

RULE offline_first
TanStack Query + IndexedDB (idb-keyval) persist the entire query cache across sessions (gcTime: 24h).
Writes are queued to IndexedDB when offline via syncQueue.ts and replayed on reconnect.
Queue uses last-write-wins: if server row is newer than mutation timestamp, mutation is discarded.

RULE offline_queue_max_retries
Failed mutations are retried up to 5 times (MAX_RETRIES). After that they are dropped with a console warning.

RULE parent_gate
Parent-only areas are gated by ParentGate component (3-second hold-to-unlock button).
Unlock session stored in localStorage with a 10-minute TTL.
Lock state is synced across tabs via StorageEvent.

RULE aac_always_free
AAC Talk Board (/kid/talk) has NO subscription check.
UpgradeBanner is NEVER shown on the talk page.
This is an explicit design constraint enforced in code.

RULE premium_gating_study
Study curriculum pages (subject/module/lesson) check useIsPremium(user?.id).
Free users see UpgradeBanner for is_premium content.
UpgradeBanner always includes a note that AAC is permanently free.

RULE kid_ui_no_paywall
Kid-facing pages show UpgradeBanner (informational) but never full payment flows.
Actual tier management is in /parent/subscription (behind ParentGate).

RULE analytics_privacy
PostHog analytics track: screen_viewed, schedule_item_completed, aac_card_tapped(category only).
AAC card labels and tts_text are NEVER sent to analytics.
Analytics are opt-out (default on). Preference in localStorage.
PostHog session recording is explicitly disabled (disable_session_recording: true).

RULE pwa
next-pwa (@ducanh2912/next-pwa) generates service worker and manifest.
PWA disabled in development (disable: NODE_ENV === 'development').
manifest.json in /public defines app name, theme color, icons.

RULE capacitor_native
Capacitor wraps the static export for Android and iOS.
Native TTS uses @capacitor-community/text-to-speech (dynamic import to avoid SSR issues).
Web TTS uses window.speechSynthesis as fallback.

RULE calm_mode
When calm_mode is true: transition-transform classes are suppressed on interactive elements.
Celebration animations are suppressed on rewards page.
NavBar applies 'calm' class to <html> element.
Break card, rewards, and all BigTileButtons respect calm prop.

RULE static_curriculum
Study curriculum data (year categories, class levels, subjects, modules, lessons, activities) is stored as static seed data in studySeedData.ts.
It is NOT fetched from the database at runtime (no study curriculum queries against DB tables).
DB tables (year_categories, class_levels, etc.) exist for future seeding/admin use.
child_progress DOES write to DB (activity completion is persisted).

DESCRIPTION A routine + AAC communication app for children with special educational needs. Designed for kids (simple, large tap targets, calm mode) managed by parents/caregivers (hold-to-unlock gate).
