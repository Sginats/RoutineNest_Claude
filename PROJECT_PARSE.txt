STEP 2 — PAGE ROUTES
════════════════════════════════════════════════════════════════════════════════

All routes are statically exported (no SSR). Every page is a "use client"
component that fetches data client-side via Supabase JS + React Query.

/
  Kid home page. Shows a 5-tile grid for: Study (📖), Talk (💬),
  Schedule (📋), Rewards (⭐), Break (🌿). Contains a subtle
  "Parent / Caregiver Sign In →" link at the bottom.
  File: src/app/page.tsx

/login
  Parent/caregiver authentication page. Supports two modes via tab toggle:
  "Sign In" (signInWithPassword) and "Create Account" (signUp). On success:
  — If user has no profiles → redirects to /onboarding.
  — If user has profiles → redirects to /parent.
  File: src/app/login/page.tsx

/onboarding
  5-step child profile setup wizard (requires auth). Steps:
  1. Child name (text input).
  2. Class level (grid of buttons from SEED_CLASS_LEVELS).
  3. Focus subjects (multi-select grid from SEED_SUBJECT_AREAS).
  4. Session length (5 / 10 / 15 / 20 min options).
  5. Learning intensity (light / medium / focused).
  On finish: creates profile row, learning plan, preferences; redirects to /parent.
  File: src/app/onboarding/page.tsx

/settings
  App settings page. Controls: calm mode toggle, grid size (2/3/4),
  sound enabled toggle, parent lock enabled toggle.
  File: src/app/settings/page.tsx

/parent
  Parent dashboard (behind ParentGate — 3-second hold-to-unlock).
  Sections: profile selector/creator, subscription status banner,
  Cards Manager (add/edit/delete AAC & task cards with IconPicker),
  Schedule Builder (create schedule, add/reorder items).
  Quick-nav cards to: Study Planner, Progress Dashboard, Rewards, Settings.
  File: src/app/parent/page.tsx

/parent/study
  Parent study planner. Shows: active learning plan (track, class level,
  intensity, session length), subject focus manager, weekly day-by-day
  schedule picker, study settings. All data stored in child_learning_plans
  and weekly_plan_entries tables.
  File: src/app/parent/study/page.tsx

/parent/study/progress
  Progress analytics dashboard. Shows per-subject completion stats, recent
  activity log, and overall streak data sourced from child_progress table.
  File: src/app/parent/study/progress/page.tsx

/parent/subscription
  Subscription management page (behind ParentGate). Shows current tier,
  tier selector cards (Free £0 / Premium £4.99/mo), feature comparison
  table, AAC-always-free reassurance panel.
  Note: payment is NOT yet wired — tier is set directly in DB.
  File: src/app/parent/subscription/page.tsx

/kid/talk
  AAC communication board. Fetches cards from DB (category="aac").
  Tapping a card speaks the tts_text (or label) aloud using tts.ts.
  Grid size configurable (2/3/4 columns). NO subscription gate — always free.
  Tracks analytics event aac_card_tapped (category only, never label).
  File: src/app/kid/talk/page.tsx

/kid/schedule
  Visual daily routine page. Loads first schedule for active profile.
  Displays items in a BigTileButton grid. Tapping an item toggles
  is_complete and awards 1 star via addRewardIfNew(). Optimistic UI.
  Offline-aware mutations via syncQueue.
  File: src/app/kid/schedule/page.tsx

/kid/rewards
  Rewards, streaks, and badges page. Shows:
  — Total star count (sum of rewards.stars).
  — Streak counter (computed from completed_at dates in child_progress).
  — 7 badges with thresholds: 1/5/10/25/50 stars + 3/7-day streaks.
  Triggers celebration animation (disabled in calm mode) when stars increase.
  File: src/app/kid/rewards/page.tsx

/kid/break
  Break time page. Cycles through 8 calming prompts every 8 seconds.
  Includes a start/pause/reset break timer. "I'm Ready!" button returns to /.
  No auth required. Reads calm mode setting.
  File: src/app/kid/break/page.tsx

/kid/study
  Study home — Today's Plan. Fetches learningPlan → weeklyEntries → filters
  SEED_SUBJECT_AREAS to today's assigned subjects. Computes per-subject
  progress percentage from child_progress. Shows StudyTile grid with
  BreakCard inserted every 3 subjects. Completion triggers celebration banner.
  File: src/app/kid/study/page.tsx

/kid/study/subject/[subjectId]
  Subject modules list. Premium gate: if !isPremium and subject.is_premium,
  shows UpgradeBanner. Otherwise lists SEED_MODULES filtered by subject.
  Each module shows completion percentage from child_progress.
  File: src/app/kid/study/subject/[subjectId]/SubjectModulesClient.tsx

/kid/study/module/[moduleId]
  Module lessons list. Premium gate on premium modules. Lists SEED_LESSONS
  filtered by module. Each shown as a LessonCard with completion status.
  File: src/app/kid/study/module/[moduleId]/ModuleLessonsClient.tsx

/kid/study/lesson/[lessonId]
  Interactive activity runner. Loads lesson → activities from SEED_LESSONS/
  SEED_ACTIVITIES. Sequentially presents each activity using the appropriate
  gameplay component. On each completion: upsertChildProgress(), addRewardIfNew().
  After all activities: shows lesson-complete celebration with earned stars.
  File: src/app/kid/study/lesson/[lessonId]/LessonActivitiesClient.tsx

/debug-env
  Developer diagnostics page. Shows whether Supabase env vars are set.
  File: src/app/debug-env/page.tsx

STEP 4 — HOOKS
════════════════════════════════════════════════════════════════════════════════

useAuth  (src/hooks/useAuth.ts)
  Manages Supabase auth session state. Subscribes to onAuthStateChange on mount;
  unsubscribes on unmount. Returns { user: User | null, loading: boolean, logout }.
  When supabase client is null (env vars missing), loading is immediately false.
  logout() calls supabase.auth.signOut().

useRequireAuth  (src/hooks/useAuth.ts)
  Wraps useAuth and adds a redirect: if !loading && !user, calls router.replace("/login").
  Returns the same { user, loading, logout } shape.
  Used on all kid and parent pages to enforce authentication.

useSettings  (src/lib/settingsHooks.ts)
  React Query wrapper for fetching settings row from DB.
  queryKey: ["settings", profileId]. Returns Settings | null.
  Only fetches when profileId is non-null.

useUpdateSettings  (src/lib/settingsHooks.ts)
  Optimistic mutation for patching settings. Uses onMutate to update cache
  immediately, then reverts on error. Calls upsertSettings(profileId, patch).
  Invalidates settings query on settle.

useSubscription  (src/lib/subscriptionHooks.ts)
  Fetches subscription tier ("free" | "premium") from profiles table via getProfile().
  queryKey: ["subscription", userId]. Defaults to "free" when no profile found.
  staleTime: 60 seconds. Only enabled when userId is non-null.

useIsPremium  (src/lib/subscriptionHooks.ts)
  Convenience wrapper around useSubscription. Returns boolean (data === "premium").
  Note: returns false while loading, so premium-gated pages should also check
  isLoading to avoid flash of upgrade banner.

useUpdateSubscription  (src/lib/subscriptionHooks.ts)
  Mutation to update subscription tier. On success: invalidates
  ["subscription", userId] and ["profiles"] queries so all dependent UI refreshes.

STEP 5 — LIBRARY MODULES
════════════════════════════════════════════════════════════════════════════════

env.ts
  Exports SUPABASE_URL and SUPABASE_ANON_KEY from process.env.
  Does NOT throw if vars are absent — allows builds without secrets.
  During static export / prerendering, both values are undefined.

supabaseClient.ts
  Creates and exports the singleton Supabase client.
  Returns null if SUPABASE_URL or SUPABASE_ANON_KEY are absent.
  Type: SupabaseClient | null.
  Every consumer must check for null before use.

db.ts
  Typed Supabase helpers for the core tables (profiles, cards, schedules,
  schedule_items, rewards, settings). All write helpers (upsert/update/delete)
  go through withOfflineQueue() which:
  — If browser is offline: enqueues a QueuedMutation in IDB and returns an
    optimistic fallback value.
  — If online: attempts the Supabase call; on network error: falls back to queue.
  Key functions:
    getProfiles(), getProfile(userId), createProfile(name)
    updateSubscriptionTier(userId, tier)
    getSettings(profileId), upsertSettings(profileId, patch)
    getCards(profileId, category?), upsertCard(card), softDeleteCard(cardId)
    getSchedules(profileId), createSchedule(profileId, title)
    getScheduleItems(scheduleId), upsertScheduleItem(item), deleteScheduleItem(id)
    updateScheduleItemDone(itemId, done)
    getRewards(profileId), addRewardIfNew(profileId, stars, reason)

studyDb.ts
  Typed Supabase helpers for study plan tables. Same withOfflineQueue pattern.
  Key functions:
    getChildLearningPlan(profileId), upsertChildLearningPlan(plan)
    getChildProgress(profileId), upsertChildProgress(progress)
    getChildPreferences(profileId), upsertChildPreferences(profileId, patch)
    getWeeklyPlanEntries(learningPlanId), replaceWeeklyPlanEntries(planId, entries)

studySeedData.ts
  Large static TypeScript file (~67 KB) containing all curriculum data.
  Exports:
    SEED_YEAR_CATEGORIES — 5 age groups (3–5, 5–7, 7–9, 9–11, 11–13).
    SEED_CLASS_LEVELS    — 8 levels (Pre-K through Grade 6).
    SEED_SUBJECT_AREAS   — 7 subjects (Communication, Reading/Language, Math,
                           Daily Living Skills, Social Skills, Fine Motor/Cognitive,
                           Creative/Calm Activities). Some flagged is_premium.
    SEED_MODULES         — Study modules per subject × class level.
    SEED_LESSONS         — Lessons within each module.
    SEED_ACTIVITIES      — Activities within each lesson (with activity_type).
  The app reads from these arrays client-side; no DB roundtrip for content.
  Note: curriculum data in the DB tables (subject_areas, lessons, etc.) mirrors
  this seed data but the app currently uses the in-memory constants directly.

studyTypes.ts
  All TypeScript types for the study system:
    ActivityType enum (9 types: visual_matching, tap_correct, sequencing,
                       listen_choose, speak_tap_aac, trace, routine_checkoff,
                       parent_guided, printable)
    StudyIntensity: "light" | "medium" | "focused"
    DayOfWeek: "monday" ... "sunday"
    Interfaces: YearCategory, ClassLevel, SubjectArea, StudyModule, Lesson,
                Activity, WeeklyPlanEntry, ChildLearningPlan, ChildProgress,
                ChildPreferences
    Insert/patch helper types for all of the above.
    Constants: YEAR_CATEGORIES, CLASS_LEVELS, SUBJECT_AREAS arrays.

types.ts
  TypeScript types for core tables:
    SubscriptionTier: "free" | "premium"
    Profile, Card, Schedule, ScheduleItem, Reward, Settings interfaces.
    CardInsert, ScheduleItemInsert, SettingsPatch, ProfilePatch helper types.

activityGameplayData.ts
  Returns interactive content data for a given activity ID + type.
  Internal tables:
    TAP_CORRECT_DATA    — per-activity choice arrays (6 activities defined).
    VISUAL_MATCHING_DATA — per-activity pair arrays (5 activities defined).
    SEQUENCING_DATA     — per-activity step arrays (3 activities defined).
  getActivityGameplayData(activityId, activityType) → ActivityGameplayData:
    { type: "tap_correct", choices: [...] }
    { type: "visual_matching", pairs: [...] }
    { type: "sequencing", steps: [...] }
    { type: "simple" }  ← fallback for unrecognised IDs or unimplemented types.

syncQueue.ts
  Offline mutation queue using idb-keyval (IndexedDB).
  Storage key: "routinenest_sync_queue".
  QueuedMutation interface: id, timestamp, table, operation, payload,
    onConflict?, matchColumn, matchValue, duplicateCheckColumns?, retries.
  Operations: "upsert" | "update" | "delete" | "insert_if_not_exists".
  MAX_RETRIES = 5. Mutations that fail 5 times are dropped.
  Key functions:
    isOnline() — SSR-safe navigator.onLine check.
    isNetworkError(err) — heuristic for fetch/network error detection.
    enqueue(mutation) — appends to queue in IDB.
    replayQueue() — processes queue FIFO against Supabase.
      Last-write-wins check: if server row updated_at > mutation timestamp, skip.
      Conflict resolution: last-write-wins only (no merge).
      _busy flag prevents concurrent replay runs.
    getQueueLength() — diagnostic count.

queryPersistence.ts
  Creates a TanStack Query IndexedDB persister.
  Storage key: "routinenest_query_cache".
  Implements Persister interface: persistClient, restoreClient, removeClient.
  Used in providers.tsx with PersistQueryClientProvider to cache all query
  results in IDB for offline read access.

profileStore.ts
  Thin localStorage wrapper for the active profile ID.
  Key: "routinenest_active_profile_id".
  Functions: getActiveProfileId(), setActiveProfileId(id), clearActiveProfileId().
  SSR-safe (returns null during server-side evaluation).

settingsHooks.ts
  React Query hooks for settings CRUD. See STEP 4 — HOOKS.

subscriptionHooks.ts
  React Query hooks for subscription tier. See STEP 4 — HOOKS.

analytics.ts

HOW THE ROUTER WORKS (LessonActivitiesClient.tsx):
  1. getActivityGameplayData(activity.id, activity.activity_type) is called.
  2. Returns { type: "tap_correct" | "visual_matching" | "sequencing" | "simple", ...data }.
  3. If type is "simple" (includes all unimplemented types), a plain instruction
     card is shown with a big "Done!" button. Tapping Done marks it complete.
  4. Interactive activities render their component and pass onComplete callback.

FALLBACK BEHAVIOUR:
  — If activity_type is unrecognised or type is "simple":
    Show: instructions text + big tappable "Done! ⭐" button.
    The child can mark any activity complete regardless of interaction type.
  — tap_correct activities with no entry in TAP_CORRECT_DATA:
    Fall back to generic 3-choice Yes/No/Maybe options.

COMPLETION FLOW:
  1. Activity component calls onComplete().
  2. LessonActivitiesClient.tsx runs:
       upsertChildProgress({ profile_id, activity_id, lesson_id, module_id,
                             completed: true, attempts: prev+1, completed_at })
       addRewardIfNew(profileId, activity.reward_points, "activity:{activityId}")
  3. showCelebration state triggers a brief celebration UI.
  4. After a break (every 3rd activity), showBreak state is set.
  5. currentIndex advances to the next activity.
  6. When all activities complete: lessonComplete state shows lesson-done screen.

STEP 9 — OFFLINE SYSTEM
════════════════════════════════════════════════════════════════════════════════

TWO LAYERS OF OFFLINE SUPPORT:

── Layer 1: Read Cache (TanStack Query + IndexedDB) ──

  Configuration (providers.tsx):
    — QueryClient with gcTime: 24 hours (keeps cached data for a full day).
    — PersistQueryClientProvider wrapping the app.
    — createIdbPersister() (queryPersistence.ts) uses idb-keyval to store the
      entire dehydrated query cache under key "routinenest_query_cache".
    — On app start, the persisted cache is restored from IDB immediately, so
      all previously fetched data (cards, schedule, study progress, etc.) is
      available offline before any network requests complete.

── Layer 2: Write Queue (syncQueue.ts) ──

  When a mutation is attempted while offline (or a network error occurs):
    1. The withOfflineQueue() helper in db.ts / studyDb.ts enqueues the
       mutation as a QueuedMutation in IDB (key: "routinenest_sync_queue").
    2. An optimistic fallback value is returned immediately so the UI updates.
    3. On next app launch or when the browser comes back online, SyncManager
       (providers.tsx) calls replayQueue().

  replayQueue() processes mutations FIFO:
    — For each mutation: fetches server row's updated_at timestamp.
    — LAST-WRITE-WINS: if server row is newer than mutation.timestamp, skip it.
    — Otherwise applies the mutation to Supabase.
    — On success: removes from queue.
    — On failure: increments retries counter. Dropped after 5 failures.
    — Stops mid-run if browser goes offline again.
    — After replay completes with n > 0 replayed: invalidates all queries
      so UI picks up authoritative server state.

  Supported offline operations: upsert, update, delete, insert_if_not_exists.
  insert_if_not_exists includes a duplicate check (used for rewards).

  CONFLICT RESOLUTION:
    Simple last-write-wins: if the server has a newer updated_at, the
    queued mutation is discarded. No field-level merging. No CRDTs.
    This is appropriate for low-conflict single-user data.

  OFFLINE INDICATOR:
    OfflineBanner component listens to window online/offline events and
    shows a gentle status bar when navigator.onLine is false.

  — Full study curriculum (all premium subjects)
  — Advanced learning plans
  — Weekly study planner
  — Detailed progress analytics

HOW AAC IS ALWAYS FREE:
  The kid/talk/page.tsx has NO subscription check. There is no call to
  useSubscription() or useIsPremium() on the talk page. AAC is structurally
  never gated and UpgradeBanner is never rendered there.
  UpgradeBanner explicitly states: "💬 Talk Board (AAC) is always free."

HOW PREMIUM GATING WORKS:
  On SubjectModulesClient, ModuleLessonsClient, LessonActivitiesClient:
    const { data: subscriptionTier, isLoading: subscriptionLoading } = useSubscription(user?.id)
    const isPremium = subscriptionTier === "premium"
    if (!subscriptionLoading && !isPremium && content.is_premium) {
      return <UpgradeBanner />
    }
  The isLoading guard prevents a flash of the upgrade banner during initial load.

SUBSCRIPTION HOOKS:
  useSubscription(userId)       → query for tier string
  useIsPremium(userId)          → boolean convenience wrapper
  useUpdateSubscription(userId) → mutation to change tier

  getProfile(userId) — reads profiles table
  updateSubscriptionTier(userId, tier) — updates profiles.subscription_tier
  Both are offline-aware (withOfflineQueue pattern).

  the subscription page UI. The /parent/subscription page shows
  "Secure payment processing coming soon" with a contact email.
